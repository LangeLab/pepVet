# Security Policy

## Reporting a vulnerability

Please do not report security issues through public GitHub issues.

Send a private report to [eneskemalergin@gmail.com](mailto:eneskemalergin@gmail.com) with:

- a short description of the issue
- affected versions or commit range if known
- reproduction steps or a proof of concept
- any suggested mitigation

You can expect an acknowledgement after the report is received, followed by a severity and remediation assessment.

## Scope

pepVet is an R package for scientific workflow planning. Relevant reports may include:

- code execution issues introduced by package behavior
- unsafe file handling or path handling bugs
- dependency or build configuration issues that materially affect users

Reports that are purely scientific disagreements, heuristic limitations, or unsupported-workflow requests should go through normal issue channels instead.
